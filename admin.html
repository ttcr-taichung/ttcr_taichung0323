GTM / UTM 部署指南
確保每個頁面都能回傳精準數據的技術清單
⚙ 部署優先順序
建議依序完成：① GTM 安裝 → ② GA4 串流 → ③ 捲動深度觸發器 → ④ UTM 規範化 → ⑤ 自訂維度。每個步驟都有獨立效益，不必等全部完成才能上線。
01
GTM 全域安裝 — 所有頁面
前往 tagmanager.google.com → 建立帳戶與容器（網頁類型）→ 取得以下兩段程式碼，分別貼入每個 HTML 頁面：
<!-- GTM Head（放在 <head> 最上方） --> <script>(function(w,d,s,l,i){w[l]=w[l]||[];w[l].push({'gtm.start': new Date().getTime(),event:'gtm.js'});var f=d.getElementsByTagName(s)[0], j=d.createElement(s),dl=l!='dataLayer'?'&l='+l:'';j.async=true;j.src= 'https://www.googletagmanager.com/gtm.js?id='+i+dl;f.parentNode.insertBefore(j,f); })(window,document,'script','dataLayer','GTM-XXXXXXX');</script>
<!-- GTM Body（放在 <body> 最上方） --> <noscript><iframe src="https://www.googletagmanager.com/ns.html?id=GTM-XXXXXXX" height="0" width="0" style="display:none;visibility:hidden"></iframe></noscript>
02
在 GTM 中建立 GA4 設定代碼
GTM 後台 → 代碼 → 新增 → 類型選「Google Analytics: GA4 設定」→ 輸入您的評估 ID（G-XXXXXXXXXX）→ 觸發條件選「All Pages」→ 發布。
<script async src="https://www.googletagmanager.com/gtag/js?id=G-J1RH1Z5918"></script> <script> window.dataLayer = window.dataLayer || []; function gtag(){dataLayer.push(arguments);} gtag('js', new Date()); gtag('config', 'G-J1RH1Z5918', { 'allow_google_signals': true, 'custom_map': { 'dimension1': 'social_platform' } }); var ref = document.referrer.toLowerCase(); var platform = '直接進站'; if (ref.indexOf('facebook.com') > -1) platform = 'Facebook'; else if (ref.indexOf('instagram.com') > -1) platform = 'Instagram'; else if (ref.indexOf('line.me') > -1) platform = 'LINE'; gtag('event', 'page_view', { 'social_platform': platform }); </script>
✓ 已設定 G-J1RH1Z5918 Google 信號已啟用 社交來源自動辨識
03
捲動深度追蹤觸發器
GTM → 觸發條件 → 新增 → 類型「捲動深度」→ 選「垂直捲動深度 %」→ 輸入 25,50,75,100 → 觸發於「所有頁面」。
再建立代碼：GA4 事件，事件名稱 scroll，附加參數 percent_scrolled。
事件名稱：scroll 參數設定： percent_scrolled → {{Scroll Depth Threshold}} page_path → {{Page Path}}
04
UTM 參數規範 — 各管道標準
每次在社群、QR Code、LINE 分享推廣時，請依下表在連結後加上 UTM 參數，後台才能區分流量來源：
推廣管道utm_sourceutm_mediumutm_content
Instagram 限動instagramsocialstory_加盟 或 story_菜單
Facebook 貼文facebooksocialpost_新品 或 post_活動
LINE 官方帳號linemessageline_vip 或 line_event
店面 QR Codeofflineqrcodestore_menu 或 store_join
加盟展活動exhibitioneventexpo_may2026
Google 廣告googlecpcad_franchise
範例完整連結（加盟頁 × IG 限動）： https://ttcr-taichung.github.io/ttcr_taichung0323/franchise.html ?utm_source=instagram &utm_medium=social &utm_campaign=加盟招募2026 &utm_content=story_join_may
05
自訂維度（Custom Dimensions）— GA4
在 GA4 → 管理 → 自訂定義 → 建立以下自訂維度，讓後台可以按頁面+訪客類型做交叉分析：
維度名稱範圍事件參數名稱用途
page_category事件page_category區分「加盟」/「菜單」/「空間」類別
user_type使用者user_type區分「一般訪客」/「潛在加盟主」
scroll_depth事件percent_scrolled記錄捲動百分比
entry_source工作階段utm_content對應 UTM content 標籤
// 在各頁面 <script> 中推送 page_category gtag('event', 'page_view', { page_category: 'franchise', // 加盟頁填 'franchise' page_title: document.title, page_path: location.pathname });
06
Google 信號啟用（年齡 / 興趣資料）
GA4 後台 → 管理 → 資料設定 → 資料收集 → 啟用「Google 信號資料收集」。啟用後，已登入 Google 帳戶的訪客的年齡、性別、興趣標籤數據將開始累積（通常需 7 天以上才有足夠樣本）。
需要：GA4 管理員權限 注意：需更新隱私政策 預計 7 天後有數據
